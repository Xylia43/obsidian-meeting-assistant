## 1. 产品概述与愿景

### 1.1 产品名称

**Obsidian Meeting Assistant**（简称 OMA）

### 1.2 一句话定义

一款 Obsidian 插件，让用户在笔记工具内完成「会议录音 → 语音转文字 → 智能会议纪要 → AI 指导建议」的全流程闭环，真正实现 **开完会即出纪要**。

### 1.3 产品愿景

> **让每一场会议的价值都被完整捕获、高效沉淀、智能激活。**

当前痛点：
- 会议中手动记笔记分散注意力，关键信息容易遗漏
- 录音文件散落在各处，事后整理成本高
- 手写纪要耗时，且格式、质量参差不齐
- 会议中产生的待办事项缺乏跟踪，容易"开完就忘"
- 现有工具（Otter.ai / Fireflies 等）独立于笔记系统之外，知识无法与个人知识库联动

OMA 的核心价值主张：
1. **零切换成本** — 在 Obsidian 内完成全部操作，录音、转写、纪要、建议一站式完成
2. **数据本地优先** — 录音文件和文本存储在用户本地 Vault，隐私可控
3. **知识网络融合** — 会议纪要天然融入 Obsidian 的双链笔记体系，与用户已有知识图谱互联
4. **AI 增强决策** — 不止记录，更提供行动建议、风险提示、待办提取

### 1.4 产品定位

| 维度 | 说明 |
|------|------|
| 平台 | Obsidian 社区插件（Desktop + Mobile） |
| 类型 | 生产力工具 / 会议管理 |
| 盈利模式 | 开源免费 + 可选付费云端转写额度 |
| 技术栈 | TypeScript + Obsidian Plugin API + Web Audio API + LLM API |

## 3. 核心用户故事

### US-01：一键录音
> 作为一名 **产品经理**，我希望在 Obsidian 中 **一键开始/暂停/停止录音**，以便 **无需切换应用即可捕获完整的会议音频**。

### US-02：实时转写
> 作为一名 **项目经理**，我希望 **录音结束后自动将语音转为文字**，以便 **快速获得可搜索、可编辑的会议文本记录**。

### US-03：智能会议纪要
> 作为一名 **研发工程师**，我希望 **一键从转写文本生成结构化会议纪要**（含议题、结论、待办），以便 **节省 30 分钟以上的手动整理时间**。

### US-04：AI 指导建议
> 作为一名 **创业者**，我希望 **AI 基于会议内容给出行动建议和风险提示**，以便 **做出更明智的决策并避免遗漏关键事项**。

### US-05：待办提取与联动
> 作为一名 **项目经理**，我希望 **AI 自动从会议内容中提取待办事项并生成 Obsidian 任务**，以便 **直接在我的任务管理流程中跟踪执行**。

### US-06：会议模板自定义
> 作为一名 **咨询顾问**，我希望 **自定义会议纪要的输出模板**（如 1-on-1、周会、头脑风暴），以便 **不同类型的会议获得最合适的纪要格式**。

### US-07：录音文件管理
> 作为一名 **重度 Obsidian 用户**，我希望 **录音文件自动保存到 Vault 指定目录，并与对应笔记双向链接**，以便 **音频和文本始终关联，随时回溯原始语境**。

### US-08：多语言支持
> 作为一名 **跨国团队成员**，我希望 **语音转文字支持中文、英文及中英混合**，以便 **准确记录双语会议的全部内容**。

### US-09：隐私与本地化
> 作为一名 **注重隐私的用户**，我希望 **所有录音和转写数据优先存储在本地**，且 **清楚知道哪些数据会发送到云端**，以便 **自主掌控个人数据安全**。

### US-10：回顾与搜索
> 作为一名 **学术研究者**，我希望 **能通过关键词搜索历史会议的转写文本和纪要**，以便 **快速定位数周前某次讨论中的具体观点**。

## 5. 非功能需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| 录音启动延迟 | ≤ 500ms（点击录音按钮到开始采集音频） |
| 转写速度（云端 Whisper） | 1 小时音频 ≤ 5 分钟完成转写 |
| 转写速度（本地 Whisper） | 1 小时音频 ≤ 15 分钟（在 M1 MacBook 基准下） |
| 纪要生成速度 | ≤ 30 秒（1 小时会议转写文本） |
| 插件加载时间 | ≤ 1 秒（不应显著影响 Obsidian 启动速度） |
| 内存占用 | 录音过程中额外内存占用 ≤ 100MB |
| 录音文件大小 | 1 小时录音 ≤ 60MB（WebM/Opus 编码） |
| 最大录音时长 | 支持连续录音 ≥ 4 小时不中断 |

### 5.2 安全与隐私

| 要求 | 说明 |
|------|------|
| 数据本地优先 | 录音文件、转写文本、会议纪要默认存储在用户本地 Vault |
| API Key 安全 | API Key 使用 Obsidian 原生设置存储机制，不明文写入笔记 |
| 网络请求透明 | 在设置中明确标注哪些功能需要网络请求（STT、LLM），用户可逐一开关 |
| 数据传输加密 | 所有 API 调用使用 HTTPS / TLS 1.2+ |
| 无遥测 | 不收集任何用户行为数据，不上传使用统计 |
| 删除可控 | 用户删除录音/笔记时，关联的所有数据完全清除 |

### 5.3 兼容性

| 维度 | 要求 |
|------|------|
| Obsidian 版本 | 兼容 Obsidian ≥ 1.4.0（Desktop）；Mobile 支持在 P1 阶段覆盖 |
| 操作系统 | macOS ≥ 12, Windows ≥ 10, Linux（Ubuntu 22.04+） |
| 移动端 | iOS ≥ 16, Android ≥ 12（P1 阶段） |
| Node.js / Electron | 兼容 Obsidian 内置的 Electron 版本 |
| 插件共存 | 不与 Templater、Dataview、Tasks、Calendar 等主流插件冲突 |
| 音频输入 | 支持系统默认麦克风、外接麦克风、蓝牙耳机麦克风 |
| STT 服务 | 支持 OpenAI Whisper API、本地 Whisper（whisper.cpp / faster-whisper） |
| LLM 服务 | 支持 OpenAI GPT-4o、Claude、Gemini、本地 Ollama 等主流模型 |

### 5.4 可用性

| 要求 | 说明 |
|------|------|
| 首次使用引导 | 插件启用后提供配置向导（选择 STT 服务 → 配置 API Key → 选择 LLM） |
| 错误提示友好 | 网络超时、API 额度不足、麦克风权限拒绝等场景提供清晰中文提示 |
| 操作可撤销 | 生成的纪要支持 Obsidian 原生 Undo 操作 |
| 快捷键 | 核心操作（开始录音、停止录音、生成纪要）均可自定义快捷键 |

## 6. MVP 范围定义

### 6.1 MVP 目标

> 用最小功能集验证核心价值假设：**Obsidian 用户是否愿意在笔记工具内完成录音→转写→纪要的全流程**。

### 6.2 MVP 包含功能

| 功能 | 范围限定 |
|------|---------|
| ✅ 会议录音（F-001） | 仅支持桌面端；WebM 格式；基础开始/停止操作 |
| ✅ 录音状态指示器（F-002） | 状态栏显示录音状态和时长 |
| ✅ 语音转文字（F-003） | 仅支持 OpenAI Whisper API；中文 + 英文 |
| ✅ 转写结果嵌入笔记（F-004） | 创建新笔记，插入转写文本 |
| ✅ 一键生成会议纪要（F-005） | 使用 OpenAI GPT-4o；固定模板（通用会议纪要） |
| ✅ AI 指导建议（F-006） | 在纪要末尾追加行动建议段落 |
| ✅ LLM 服务配置（F-007） | 仅支持 OpenAI API Key 配置 |
| ✅ STT 服务配置（F-008） | 仅支持 OpenAI Whisper API Key 配置 |

### 6.3 MVP 明确排除

| 排除项 | 理由 |
|--------|------|
| ❌ 自定义模板 | MVP 使用固定模板，降低开发复杂度 |
| ❌ 本地 Whisper | 需要额外的本地模型管理，MVP 优先跑通云端链路 |
| ❌ 移动端 | Mobile Audio API 差异大，MVP 专注桌面端 |
| ❌ 实时转写 | 技术复杂度高，MVP 采用录音后转写 |
| ❌ 说话人识别 | 依赖额外模型，非核心价值验证所必需 |
| ❌ 多 LLM 支持 | MVP 仅支持 OpenAI，后续迭代扩展 |

### 6.4 MVP 成功指标

| 指标 | 目标值 |
|------|--------|
| 插件安装量 | 发布 30 天内 ≥ 500 |
| 用户留存（7日） | ≥ 30%（基于 GitHub issue / 社区反馈间接评估） |
| GitHub Stars | 发布 30 天内 ≥ 200 |
| 社区反馈情感 | 正面评价 ≥ 70% |
| 关键 Bug | P0 Bug 0 个（发布后 7 天内） |

### 6.5 MVP 时间线

| 阶段 | 时间 | 产出 |
|------|------|------|
| 设计与技术选型 | 第 1-2 周 | 技术方案文档、UI 原型 |
| 核心开发 | 第 3-6 周 | 录音 + STT + 纪要生成 |
| 联调与测试 | 第 7-8 周 | 端到端测试、性能优化 |
| 内测 | 第 9 周 | 邀请 20-30 名 Obsidian 用户内测 |
| 公开发布 | 第 10 周 | 提交 Obsidian 社区插件市场 |

## 7. 竞品分析

### 7.1 竞品概览

| 维度 | Audio Notes (Obsidian) | Otter.ai | Fireflies.ai | Notta | **OMA（本产品）** |

| 智能纪要 | ❌ | ✅（基础） | ✅（详细） | ✅ | ✅（可自定义） |
| AI 建议 | ❌ | ❌ | ✅（有限） | ❌ | ✅（深度建议） |
| 知识库融合 | ✅（Obsidian 原生） | ❌ | ❌ | ❌ | ✅（双链 + 图谱） |

| 离线能力 | ✅（本地 Whisper） | ❌ | ❌ | ❌ | ✅（P2 阶段） |

### 7.2 各竞品深度分析

#### Audio Notes（Obsidian 插件）

- **优势：** 与 Obsidian 深度集成；支持本地 Whisper 转写；社区口碑好
- **不足：** 仅做录音 + 转写，不提供智能纪要和 AI 建议；缺少结构化输出
- **启示：** 证明了 Obsidian 内录音转写的需求真实存在，OMA 在此基础上做 **纪要 + AI** 的增值

#### Otter.ai

- **优势：** 实时转写体验优秀；说话人识别准确；团队协作功能完善
- **不足：** 数据完全云端，隐私风险；与个人笔记系统割裂；中文支持弱；价格偏高
- **启示：** 实时转写和说话人识别是高价值功能，OMA 可在 P2 阶段跟进

#### Fireflies.ai

- **优势：** 自动加入在线会议（Zoom/Meet/Teams）；AI 摘要质量高；丰富的集成
- **不足：** 侧重在线会议场景，不适合线下会议；数据云端存储；价格高
- **启示：** 会议 bot 自动加入模式值得关注，但 OMA 定位为本地工具，更适合线下场景

#### Notta

- **优势：** 多语言支持好（尤其中日韩）；实时转写 + 编辑；价格相对合理
- **不足：** 独立应用，不与笔记系统融合；导出格式有限
- **启示：** 多语言（尤其中文）是 OMA 的差异化优势之一，要做到最好

### 7.3 OMA 差异化定位

```
                    智能程度（AI 纪要 + 建议）
                         ↑
                         │
        Fireflies ●      │      ● OMA（目标位置）
                         │
           Otter ●       │
                         │
           Notta ●       │      ● Audio Notes + AI
                         │
        ─────────────────┼──────────────────→ 知识库融合度
                         │
                    (独立应用)          (Obsidian 原生)
```

**OMA 的核心差异化：**
1. **唯一同时具备 AI 纪要 + Obsidian 原生融合** 的产品
2. **本地优先 + 开源免费**，对隐私敏感用户极具吸引力
3. **中文场景优化**，服务中文 Obsidian 社区（全球第二大 Obsidian 用户群体）

## 8. 验收标准

### 8.1 功能验收标准

#### AC-001：会议录音
- [ ] 用户可通过工具栏图标或命令面板（`Ctrl/Cmd+P → Meeting Assistant: Start Recording`）启动录音
- [ ] 录音开始后，状态栏显示 🔴 录音指示器及已录时长（格式：`MM:SS`）
- [ ] 用户可暂停和恢复录音，暂停期间不采集音频
- [ ] 用户可停止录音，录音文件自动保存到配置的 Vault 目录（默认 `attachments/meetings/`）
- [ ] 录音文件格式为 WebM（Opus 编码），文件名格式 `meeting-YYYY-MM-DD-HHmmss.webm`
- [ ] 录音期间关闭 Obsidian 会弹出确认对话框，防止意外丢失

#### AC-002：语音转文字
- [ ] 录音停止后，自动弹出提示询问是否开始转写
- [ ] 转写过程中显示进度指示（转写中... / 已完成 XX%）
- [ ] 转写完成后，自动创建新笔记或在当前笔记中插入转写文本
- [ ] 转写文本包含时间戳标记（每 30 秒一个锚点），格式 `[00:30]`
- [ ] 中文语音转写准确率 ≥ 90%（标准普通话、安静环境）
- [ ] 英文语音转写准确率 ≥ 93%（标准口音、安静环境）
- [ ] API 调用失败时显示友好错误提示，支持重试

#### AC-003：一键生成会议纪要
- [ ] 转写完成后，用户可通过按钮或命令 `Meeting Assistant: Generate Summary` 一键生成纪要
- [ ] 纪要包含以下结构化段落：会议基本信息、议题列表、讨论要点（按议题分组）、决策事项、待办事项（含负责人如有提及）
- [ ] 纪要以 Markdown 格式输出，与 Obsidian 渲染兼容
- [ ] 生成过程 ≤ 30 秒（1 小时会议转写文本）
- [ ] 生成过程中显示加载状态

#### AC-004：AI 指导建议
- [ ] 纪要生成后，自动追加「💡 AI 建议」段落
- [ ] 建议内容包含：后续行动建议（2-5 条）、潜在风险提示、关注事项
- [ ] 建议内容与会议上下文相关，非泛泛而谈
- [ ] 用户可选择不生成建议（设置中可关闭）

#### AC-005：配置与设置
- [ ] 插件设置页包含：STT 服务选择、STT API Key 输入、LLM 服务选择、LLM API Key 输入、录音存储路径、纪要输出路径、默认语言
- [ ] API Key 输入框为密码类型（不明文显示）
- [ ] 提供 "Test Connection" 按钮验证 API Key 有效性
- [ ] 设置变更即时生效，无需重启 Obsidian
